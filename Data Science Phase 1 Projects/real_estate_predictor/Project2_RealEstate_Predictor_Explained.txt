WHAT IS THIS PROJECT?
---------------------
We built a Machine Learning model that looks at real Boston
housing data and predicts the price of a house based on its
features like number of rooms, crime rate, neighbourhood quality
and more. This is a REGRESSION problem — we predict an actual
number (price) instead of a category (yes/no).

Dataset  : Boston Housing Dataset
Source   : Kaggle (vikrishnan/boston-house-prices)
File     : housing.csv
Houses   : 506 real properties
Features : 13 measurements per house
Target   : MEDV — Median House Value in $1000s
Result   : 88.3% R² Score (Decision Tree best model)

KEY DIFFERENCE FROM PROJECT 1:
    Project 1 = Classification → predict YES or NO (planet/no planet)
    Project 2 = Regression     → predict a NUMBER (house price)
    These are the 2 main types of Supervised Learning!

    from sklearn.tree import DecisionTreeRegressor
    from sklearn.ensemble import RandomForestRegressor

WHAT EACH NEW IMPORT DOES:
---------------------------

PolynomialFeatures
    - Creates new features by combining existing ones
    - degree=2 creates: x, x², x*y, y², etc.
    - Allows the model to fit CURVED relationships (not just straight lines)
    - Example: price vs rooms is not perfectly linear — it curves!

Pipeline
    - Chains multiple steps together into one object
    - We use it to combine PolynomialFeatures + LinearRegression
    - Benefit: can treat them as a single model for cross-validation
    - Makes code cleaner and prevents data leakage

mean_squared_error (MSE)
    - Average of (predicted - actual)² for all houses
    - Squares the errors so big mistakes are penalised heavily
    - We take the square root to get RMSE (same units as price)

mean_absolute_error (MAE)
    - Average of |predicted - actual| for all houses
    - More intuitive: "on average we are $X wrong"
    - Less sensitive to outliers than MSE

r2_score (R² Score)
    - How much of the price variance our model explains
    - 1.0 = perfect, 0.0 = no better than guessing the mean
    - 0.8834 means we explain 88.34% of price differences

DecisionTreeRegressor
    - Same idea as DecisionTreeClassifier but for regression
    - Instead of predicting a class, predicts a number
    - Each leaf node returns the average price of training houses
      that fell into that leaf

RandomForestRegressor
    - Imported but not used as main model (available for extension)
    - Combines many Decision Trees for even better results
    - This is Phase 2 territory — Ensemble Methods!

CELL 5 — TRAINING 3 REGRESSION MODELS
================================================================

CODE:
    models = {
        'Linear Regression'    : LinearRegression(),
        'Polynomial Regression': Pipeline([
            ('poly',   PolynomialFeatures(degree=2)),
            ('linear', LinearRegression())
        ]),
        'Decision Tree': DecisionTreeRegressor(max_depth=5, random_state=42),
    }

WHAT EACH MODEL DOES:
----------------------

--- MODEL 1: LINEAR REGRESSION ---
LinearRegression()

    What it is:
        Fits a straight line (or hyperplane in multiple dimensions)
        through the data to predict house prices
        Formula: price = w1*CRIM + w2*RM + w3*LSTAT + ... + bias

    How it learns:
        Finds the weights (w1, w2...) that minimise MSE
        Uses the Ordinary Least Squares (OLS) method
        Closed-form solution — no iteration needed, very fast!

    Limitation:
        Can only model LINEAR relationships
        If price and rooms have a curved relationship → Linear fails

    RESULT: R² = 0.6688 (explains 66.9% of variance)
    Translation: Decent but misses complex patterns

--- MODEL 2: POLYNOMIAL REGRESSION ---
Pipeline([
    ('poly',   PolynomialFeatures(degree=2)),
    ('linear', LinearRegression())
])

    What it is:
        Extends Linear Regression to fit CURVED relationships
        Creates new features: RM², CRIM², RM×LSTAT, etc.
        Then fits a linear model on these new features

    PolynomialFeatures(degree=2)
        - Takes original 13 features
        - Creates all combinations up to degree 2
        - Results in many more features (13 → 105 features!)
        - This allows curved fit without changing the linear model

    Pipeline
        - Step 1: Transform features (add polynomial terms)
        - Step 2: Fit Linear Regression on transformed features
        - Treated as one model — clean and prevents data leakage

    RESULT: R² = 0.8056 (explains 80.6% of variance)
    Translation: Much better! Curved relationships captured

--- MODEL 3: DECISION TREE REGRESSOR ---
DecisionTreeRegressor(max_depth=5, random_state=42)

    What it is:
        Makes predictions by asking a series of YES/NO questions
        Each leaf node returns the AVERAGE price of training houses
        that ended up in that leaf after all the splits

    How it differs from classifier version:
        Instead of majority vote → takes average of target values
        Splits are chosen to minimise MSE (not Gini impurity)

    max_depth=5
        - Maximum 5 levels of questions
        - Prevents overfitting
        - Without limit: tree memorises every house → fails on new data

    Why it won:
        Captures non-linear AND interaction effects between features
        Example: "If RM > 7 AND LSTAT < 5 AND CRIM < 0.1 → very expensive"
        Linear models cannot capture these 3-way interactions!

    RESULT: R² = 0.8834 (explains 88.3% of variance)
    Translation: Best model! Captures complex price patterns

THE 3 REGRESSION METRICS EXPLAINED:

    R² Score (R-squared)
        Formula: 1 - (SS_residual / SS_total)
        Range: 0.0 to 1.0 (can be negative for very bad models)
        Meaning: What % of price variation does model explain?
        Our best: 0.8834 = 88.34% explained ✅

    MAE (Mean Absolute Error)
        Formula: average(|predicted - actual|)
        Meaning: On average, predictions are off by $X
        More intuitive for business understanding
        Example: MAE=$3,000 means predictions off by $3k on average

    RMSE (Root Mean Squared Error)
        Formula: sqrt(average((predicted - actual)²))
        Meaning: Similar to MAE but penalises big errors more
        Always >= MAE
        Use when large errors are especially costly

Cross-Validation (cv=5, scoring='r2')
    - Same as Project 1 but scoring R² instead of F1
    - Trains 5 times on different portions of training data
    - Gives reliable estimate of real-world performance
    - cv_r2 ± cv_std shows consistency of the model

CELL 8 — HOUSE PRICE PREDICTOR TOOL
================================================================

CODE:
    def predict_house_price(crim, rm, lstat, ...):
        house = medians.copy()
        house['CRIM']  = crim
        house['RM']    = rm
        house['LSTAT'] = lstat
        house_scaled = scaler.transform([house])
        price = models['Decision Tree'].predict(house_scaled)[0]
        return price

WHAT IT DOES:
--------------

def predict_house_price(...)
    - Creates a reusable function to predict any house price
    - Takes key inputs: crime rate, rooms, poverty %
    - This makes the project PRODUCT-LIKE — a real usable tool!

medians.copy()
    - Gets median values of all 13 features as baseline
    - For features we don't specify, uses typical Boston values
    - This way user only needs to specify the most important features

scaler.transform([house])
    - Scales the input using the SAME scaler fitted on training data
    - This is critical — must use same scaling for predictions to work!
    - [house] wraps it in a list because scaler expects 2D input

models['Decision Tree'].predict(house_scaled)[0]
    - Uses our best trained model to make the prediction
    - [0] takes the first (and only) prediction from the array

PREDICTION RESULTS:
    🏚️  Cheap House    (crime=10, rooms=4.5, lstat=25) → $13,170
    🏠  Average House  (crime=0.5, rooms=6.0, lstat=12) → $20,356
    🏡  Expensive House(crime=0.02, rooms=8.0, lstat=3) → $35,200

INSIGHT:
    Expensive house is 2.7x the price of cheap house!
    This matches real estate market patterns perfectly.
    The model learned true economic relationships from data.

KEY CONCEPTS USED IN THIS PROJECT
================================================================

1. REGRESSION vs CLASSIFICATION
   Regression  = predict a number (price, temperature, salary)
   Classification = predict a category (yes/no, spam/not spam)

2. LINEAR REGRESSION
   Fits a straight line through data
   Fast, interpretable, but limited to linear relationships

3. POLYNOMIAL REGRESSION
   Extends linear to fit curves
   More powerful but risk of overfitting at high degrees

4. DECISION TREE REGRESSOR
   Asks yes/no questions to navigate to a prediction
   Captures non-linear and interaction effects
   Best performer here at 88.3% R²

5. PIPELINE
   Chains preprocessing + model into one object
   Prevents data leakage in cross-validation

6. R² SCORE
   Primary metric for regression
   How much variance in price our model explains

7. MAE & RMSE
   How far off predictions are in dollar terms
   MAE more intuitive, RMSE penalises large errors more

8. FEATURE IMPORTANCE
   Which measurements matter most for predictions
   Gives real-world insights, not just numbers

9. ACTUAL VS PREDICTED PLOT
   Visual confirmation of model quality
   Dots near diagonal line = good predictions
