WHAT IS THIS PROJECT?
---------------------
We built an Unsupervised Machine Learning model that analyses
real Sloan Digital Sky Survey (SDSS) data and automatically
groups 100,000 space objects into clusters — WITHOUT being
told what they are! The model discovers that stars, galaxies
and quasars naturally form distinct groups in data space.

Dataset  : Sloan Digital Sky Survey (SDSS17)
Source   : Kaggle (fedesoriano/stellar-classification-dataset-sdss17)
File     : star_classification.csv
Objects  : 100,000 real space objects
Features : 8 astronomical measurements
Classes  : GALAXY (59.4%), STAR (21.6%), QSO (19.0%)
Method   : K-Means Clustering + PCA Visualisation

KEY DIFFERENCE FROM ALL PREVIOUS PROJECTS:
    Projects 1-4 = SUPERVISED Learning
                   → Model learns from labelled data
                   → We tell it the right answers
                   → It learns to predict labels

    Project 5    = UNSUPERVISED Learning
                   → No labels, no right answers given
                   → Model finds hidden patterns itself
                   → It discovers natural groupings!

    This is fundamentally different — the model has never
    seen "GALAXY", "STAR", "QSO" labels. It just looks at
    the numbers and finds structure on its own!

WHAT IS REDSHIFT?
    When objects move away from us, light waves stretch
    Like an ambulance siren getting lower as it passes
    Higher redshift = moving faster away = farther away
    QSOs are so far away their light is highly redshifted
    Stars in our galaxy have nearly zero redshift!

CELL 4 — SCALING & PCA
================================================================

CODE:
    scaler  = StandardScaler()
    X_scaled = scaler.fit_transform(X)
    pca     = PCA(n_components=3)
    X_pca   = pca.fit_transform(X_scaled)

WHY SCALE BEFORE PCA?
    PCA finds directions of maximum variance
    If one feature has values 0-1000 and another 0-1,
    the large-scale feature dominates unfairly
    Scaling ensures all features contribute equally

HOW PCA WORKS (step by step):
    Step 1: Centre the data (subtract mean)
    Step 2: Calculate covariance matrix
    Step 3: Find eigenvectors (principal components)
    Step 4: PC1 = direction of maximum variance
    Step 5: PC2 = direction of 2nd most variance (perpendicular)
    Step 6: PC3 = direction of 3rd most variance
    Step 7: Project all data onto these 3 directions

PCA RESULTS:
    Original features : 8
    PCA components    : 3
    Total variance kept: 80.42%

    This means we compressed 8 dimensions into 3
    while keeping 80.42% of all information!
    The remaining 19.58% is lost but it's the least
    important variation in the data.

WHY n_components=3?
    2 components → can plot in 2D (flat chart)
    3 components → can plot in 3D (space-like chart)
    We chose 3 to get both 2D and 3D visualisations!

    kmeans.fit(X_clean)

OUTLIER REMOVAL:
    stats.zscore(X_scaled)
        → Calculates how many standard deviations each
          value is from the mean
        → Values beyond 3 std = extreme outliers
        → 2,116 extreme outliers removed (2.1%)
        → Without this: 1 outlier dominated a whole cluster!

    Why outliers cause problems in K-Means:
        K-Means minimises distance to cluster centres
        One extreme point forces a centroid to sit near it
        Entire cluster = 1 outlier, rest get 2 clusters
        This is exactly what happened initially!

THE ELBOW METHOD:
    Run K-Means for K=2,3,4,5,6
    Plot inertia (total distance from centroids) vs K
    Inertia always decreases as K increases
    "Elbow" = point where adding more clusters stops helping much

    Our results:
    K=2: 30,436  K=3: 24,300  K=4: 20,047  K=5: 17,510

    The curve bends at K=3 confirming 3 natural groups!
    This matches our 3 actual classes perfectly!

SILHOUETTE SCORE:
    Measures cluster quality (-1 to +1)
    For each point: how similar to own cluster vs others?
    0.28-0.31 range = moderate separation
    Space data naturally overlaps somewhat → expected!

K-MEANS ALGORITHM (step by step):
    Step 1: Place 3 centroids randomly
    Step 2: Assign each point to nearest centroid
    Step 3: Move centroid to mean of its assigned points
    Step 4: Repeat until centroids stop moving
    n_init=10: Run 10 times, keep best result
    max_iter=300: Maximum 300 iterations per run

CLUSTER RESULTS:
    Cluster 0: 31,880 objects (32.6%)
        → Galaxy-dominated (64.6%)
        → Contains most of the stars too (33.3%)

    Cluster 1: 44,377 objects (45.3%)
        → Mixed — Galaxies (56.6%) + QSOs (28.5%)
        → Largest cluster

    Cluster 2: 21,627 objects (22.1%)
        → Galaxy-dominated (62.5%)
        → Most balanced mix of all 3 types

KEY CONCEPTS USED IN THIS PROJECT
================================================================

1. UNSUPERVISED LEARNING
   No labels provided to the model
   Finds hidden patterns by itself
   K-Means, Hierarchical Clustering, PCA

2. K-MEANS CLUSTERING
   Partitions data into K groups
   Minimises within-cluster distances
   Requires choosing K in advance

3. ELBOW METHOD
   Visual technique to choose optimal K
   Plot inertia vs K, find the "elbow" bend
   K=3 confirmed for our dataset

4. SILHOUETTE SCORE
   Measures cluster separation quality
   0.28-0.31 = moderate (space data overlaps)

5. PCA (Principal Component Analysis)
   Reduces 8 features to 3 components
   Keeps 80.42% of information
   Enables 2D and 3D visualisation

6. OUTLIER REMOVAL (Z-score method)
   Z-score > 3 = extreme outlier
   2,116 removed to prevent cluster distortion
   Critical for K-Means stability

7. FEATURE IMPORTANCE (Random Forest)
   Switch to supervised to analyse features
   Redshift = 59.78% dominant feature
   Sky position = barely matters

8. 3D VISUALISATION
   projection='3d' in matplotlib
   Adds depth to space visualisation
   Literally looks like the night sky!
